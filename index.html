Chapter 01

The origin arc.

Not a ladder of job titles. More like four worlds that kept teaching me a different part of the craft.

If I can’t explain a system clearly, I probably don’t understand it yet.

That lesson began in a classroom and followed me into every codebase after it.

01
Origins · Java + teaching

Learn it. Build it. Teach it.

I started as a Java developer and trainer, helping more than 100 students move from “what is this?” to working software. Teaching made clarity a permanent part of my engineering style.

02
Mobile saga · Media + retail + healthcare

Software left the desk and entered people’s hands.

Streaming screens, shopping flows, payments, offline behaviour, 2FA, clinician messaging—Android taught me that every backend decision eventually becomes a human experience.

03
Enterprise arc · Banking + insurance

The systems got heavier. The thinking got calmer.

At Infosys, the quests became Java APIs, cloud migration, Linux deployments, secure credentials, production defects, and files measured in gigabytes—not megabytes.

∞
Now · Backend + DevOps + creator mode

Still levelling up—with receipts.

I completed an M.Sc. in Computer Science, keep building hands-on backend labs, study system design and DevOps, and publish the explanations I wish I had when I began.

Chapter 02

Boss fights, not bullet points.

The moments I remember are rarely the easy releases. They’re the problems that initially looked unreasonable.

Encounter // Memory DragonCritical

Gigabyte files were eating the heap.

Recurring failures appeared when 3–5 GB JSON files were loaded like ordinary payloads. The move was to stop holding the whole world in memory and rebuild the path around streaming.

Special move: streaming JSON
3–5 GBProcessed reliably
Encounter // Legacy HydraMulti-head

SOAP/XML in. REST/JSON out.

Modernizing interfaces without disrespecting the systems already depending on them. Seven to eight Java endpoints helped bridge the old contracts and the new world.

Special move: careful evolution
7–8REST endpoints
Encounter // Cloud CastleMigration

Move the castle while people still live in it.

More than 50 customer workflows moved to OpenText Cloud. The real weapon was not a shiny tool—it was patient coordination, validation, and controlled delivery.

Special move: zero-drama cutover

Encounter // Party RaidCo-op

One product. Many teams. No solo hero fantasy.

Healthcare delivery meant aligning mobile, backend, product, and testing while shipping login, 2FA/SSO, messaging, fixes, releases, and code reviews. Good delivery is a multiplayer game.

Special move: translate between teams
